## 1. Root Cause Analysis

V2.7 Pipeline Dashboard 虽然功能完善，但在数据安全性、持久化、可恢复性方面存在风险：

- 页面关闭/浏览器崩溃可能导致状态丢失
- Prompt 历史仅保存在组件状态，刷新丢失
- 资源可能仅保存 URL（URL 过期后不可用）
- clearAll() 一键清理风险过高
- 缺少备份和恢复机制
- 缺少故障恢复能力

## 8. Remaining Risks

1. **IndexedDB 配额**: 浏览器 IndexedDB 有存储限制（通常 ~500MB-1GB），长时间运行后需监控
2. **视频二进制大文件**: 大视频 Blob 可能导致 IndexedDB 写入性能问题，建议增加压缩
3. **恢复后 sceneId 映射**: 恢复时间轴时，clip 的 source.shotId 依赖原 shot ID 映射
4. **自动保存覆盖**: 30s 自动保存间隔 - 如需更短间隔可调整
5. **Prompt 版本去重**: 当前按全字符串匹配去重，语义相同但格式不同的 prompt 不会去重
6. **AssetIntegrityService 图片验证**: 使用 Image() 加载验证，5s 超时较大文件可能卡顿

## 9. V3.0 Recommendation

1. **Story → Final Video 全自动流水线**: 输入故事 → 角色生成 → 分镜 → Prompt → 图片 → 视频 → 字幕 → 封面 → 合成
2. **云端任务调度**: 基于队列的远程任务分发 + 轮询状态
3. **多项目队列管理**: 同时管理多个项目的生产队列，优先级调度
4. **AI 字幕生成**: 自动语音识别 + 字幕合成
5. **资源去重**: 基于内容 hash 的 IndexedDB 去重存储
6. **Service Worker 缓存**: 离线时仍可访问已缓存的资源和页面
